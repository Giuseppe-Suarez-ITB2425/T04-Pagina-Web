Registrador de incidencias

Si estais aqui es porque quereis saber mas sobre el registrador de incidencias, en este proyecto realizado en la asignatura MDS con la profesora Meritxell Durany. Hemos hecho formularios, usado en PyCharm, github, entre otros programas, donde ahora profundicaremos en el siguiente apartado. :D

Información del proyecto

Primero realizamos un formulario de Google donde hemos tenido que asignar diferentes preguntas como pueden ser: (¿Dónde ocurrió? Breve explicación del problema, ¿hubo algún aviso antes del fallo?) Entre otras preguntas más... Después tuvimos nosotros mismos que rellenar el formulario una cierta cantidad de veces, luego tuvimos que descargarlo en un cierto formato que era CSV. Por otro lado, después de rellenar el formulario y exportarlo en el formato CSV, debimos migrar estos datos a XML, un lenguaje de programación. Luego tuvimos que realizar, con código Python (otro tipo de lenguaje de programación), un proceso para que este formato XML se pudiera imprimir o mostrar por pantalla. Para finalizar, tuvimos que hacer que los datos se pudieran almacenar en un formato llamado JSON, que es un formato legible para los humanos y analizable o entendible para el equipo.

Conclusión del proyecto

En conclusión, hemos aprendido código o mejorado los conocimientos que ya teníamos de Python, además de ser nuestro primer proyecto en grupo de la asignatura. Por otro lado, hemos tocado formatos que, a lo personal, si había escuchado, sabía más o menos lo que era, pero nunca había trabajado con ellos. La verdad, ha sido divertido adquirir conocimientos nuevos. Tocando formatos como JSON, XML y Python ha sido una nueva experiencia; poder exportar archivos y ver cómo los archivos no eran legibles para nosotros y transformarlos ha sido un proceso un poco complicado, pero cuando se logró fue genial. Una práctica muy completa.
